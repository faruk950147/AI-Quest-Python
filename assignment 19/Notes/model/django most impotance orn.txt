Django Query Optimization কী?

Database query কমানো + efficient করা = Query Optimization

Goal:

কম query execute করা
fast response পাওয়া
server load কমানো
Common Problem (N+1 Query Problem)
products = Product.objects.all()

for product in products:
    print(product.category.name)

Problem:

1 query → products
N query → category

Total = N+1 queries 

Solution 1: select_related() (ForeignKey)
products = Product.objects.select_related('category')

for product in products:
    print(product.category.name)

✔️ 1 query তেই সব data

Use when:

ForeignKey
OneToOneField
Solution 2: prefetch_related() (Many-to-Many / Reverse)
products = Product.objects.prefetch_related('tags')

✔️ separate query but optimized
Use when:

ManyToMany
reverse FK
Difference
Method	Use case	Query
select_related	FK, OneToOne	JOIN
prefetch_related	M2M, reverse	separate query
Only প্রয়োজনীয় data আনো (only, values)
products = Product.objects.only('id', 'name')

or

products = Product.objects.values('id', 'name')

✔️ unnecessary field load হয় না

.exists() ব্যবহার করো (faster than count)

    print("Found")

avoid:

Product.objects.filter(price=100).count()
.iterator() (Large Data)

Aggregation ব্যবহার করো

✔️ Python loop avoid

Index ব্যবহার করো

1. Model Optimization (Best Structure)
class Category(models.Model):
    name = models.CharField(max_length=100, db_index=True)


class Product(models.Model):
    category = models.ForeignKey(Category, on_delete=models.CASCADE)
    name = models.CharField(max_length=200, db_index=True)
    price = models.DecimalField(max_digits=10, decimal_places=2, db_index=True)
    created_at = models.DateTimeField(auto_now_add=True)


class Variant(models.Model):
    product = models.ForeignKey(Product, on_delete=models.CASCADE, related_name='variants')
    size = models.CharField(max_length=50)
    color = models.CharField(max_length=50)
    stock = models.IntegerField(default=0)

✔️ db_index=True → search fast
✔️ related_name='variants' → clean query

2. BAD View (Slow)
def product_list(request):
    products = Product.objects.all()

    data = []
    for p in products:
        variants = Variant.objects.filter(product=p)
        data.append({
            'name': p.name,
            'category': p.category.name,
            'variants': variants
        })

    return render(request, 'shop.html', {'data': data})

Problem:

N+1 query
loop এর ভিতরে query
3. Optimized View (FAST ⚡)
def product_list(request):
    products = Product.objects.select_related('category') \
        .prefetch_related('variants') \
        .only('id', 'name', 'price', 'category__name')

    return render(request, 'shop.html', {'products': products})

✔️ 1–2 query only
✔️ super fast

{% endfor %}

10. Caching (Huge boost)
from django.core.cache import cache

def product_list(request):
    products = cache.get('product_list')

    if not products:
        products = Product.objects.select_related('category') \
            .prefetch_related('variants')
        cache.set('product_list', products, 60)

    return render(request, 'shop.html', {'products': products})
🚀 11. Pagination (Must for large data)
from django.core.paginator import Paginator

def product_list(request):
    products = Product.objects.all()
    paginator = Paginator(products, 10)

    page = request.GET.get('page')
    products = paginator.get_page(page)

    return render(request, 'shop.html', {'products': products})
