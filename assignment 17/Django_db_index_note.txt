Django db_index=True Note

1.  db_index=True কি? db_index=True দিলে Django database এ ওই field এর
    উপর index তৈরি করে। Index থাকলে filter/search দ্রুত হয়।

2.  কখন ব্যবহার করবো?

-   যেসব field দিয়ে বেশি filter/search করা হয়
-   slug field (product detail page এর জন্য)
-   created_at (recent data show করার জন্য)
-   is_active (active product filter করার জন্য)
-   login field (username/email)

3.  কখন ব্যবহার করবো না?

-   বড় TextField (description)
-   যেসব field কখনো filter করা হয় না
-   unique=True দিলে আলাদা করে db_index দরকার নেই
-   ForeignKey field এ Django automatically index দেয়

4.  সতর্কতা:

-   বেশি index দিলে database size বাড়ে
-   insert/update একটু slow হতে পারে

5.  উদাহরণ:

class Product(models.Model): name = models.CharField(max_length=200)
slug = models.SlugField(unique=True) price =
models.DecimalField(max_digits=10, decimal_places=2) created_at =
models.DateTimeField(auto_now_add=True, db_index=True) is_active =
models.BooleanField(db_index=True)

সংক্ষেপে: যে field দিয়ে বেশি query হবে, সেখানে db_index ব্যবহার করবো।
